# 安全策略

## 支持范围

- `main` 分支和最新稳定 Release：接受安全修复。
- `develop` 和开发预览 Release：接受报告，但可能随开发版本一起修复。
- 更早的历史版本：通常不再单独维护。

## 报告安全问题

请使用 GitHub 仓库 Security 页面中的 **Report a vulnerability** 私下提交。不要用公开 Issue 报告以下内容：

- Token、Cookie、账号信息或其他凭证泄露。
- 可导致本地文件越权读取、覆盖或命令执行的问题。
- 可能暴露私人反馈、聊天内容或本地媒体文件的问题。
- GitHub 反馈晋升、来源验证或发布链路的绕过方式。

报告中请提供受影响版本、复现步骤、实际影响和建议修复方向。请删除真实凭证和不必要的个人信息。

维护者会确认已收到报告，并在完成初步判断后说明后续处理方式。修复公开前，请不要披露可直接利用的细节。
